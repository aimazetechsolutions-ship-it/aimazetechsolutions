VIDEO BACKGROUND CONTROL GUIDE

Why the same homepage video was playing everywhere:
The previous shared.js accepted only plain 11-character YouTube video IDs in content.json.
If you pasted a full YouTube URL or iframe embed code for about/services/odoo/etc., the script treated it as invalid and fell back to the homepage video.

Fixed in v7:
- You can now paste either:
  1) YouTube video ID only, for example: QyhwSYhX09s
  2) Full YouTube URL, for example: https://www.youtube.com/watch?v=QyhwSYhX09s
  3) Short URL, for example: https://youtu.be/QyhwSYhX09s
  4) Embed URL, for example: https://www.youtube.com/embed/QyhwSYhX09s
  5) Full iframe embed code

Where to change videos:
Open content.json and update:

"hero": {
  "videoUrl": "YOUR_HOME_PAGE_VIDEO_ID_OR_URL"
}

"pageVideos": {
  "about": "YOUR_ABOUT_VIDEO_ID_OR_URL",
  "services": "YOUR_SERVICES_VIDEO_ID_OR_URL",
  "odoo": "YOUR_ODOO_VIDEO_ID_OR_URL",
  "industries": "YOUR_INDUSTRIES_VIDEO_ID_OR_URL",
  "portfolio": "YOUR_PORTFOLIO_VIDEO_ID_OR_URL",
  "blog": "YOUR_BLOG_VIDEO_ID_OR_URL",
  "contact": "YOUR_CONTACT_VIDEO_ID_OR_URL"
}

Important:
- After uploading new files, clear browser cache or open in Incognito.
- On Netlify, redeploy the full folder/ZIP.
- Keep valid JSON formatting if editing content.json.
